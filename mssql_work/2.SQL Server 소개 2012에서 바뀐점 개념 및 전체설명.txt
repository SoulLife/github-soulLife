Section 02 : SQL Server 소개

마이크로소프트는 2012년 3월에 SQL Server  2012 RTM(정식) 버전을 출시하였다. 


*여기서 잠깐
소프트웨어가 출시되기까지 다양한 버전에 대해 정리하고 넘어가자

*Alpha : 개발 직후의 버전으로 상당히 많은 버그를 포함할수 있다. 대개 회사 내부에서 사용된다. 

*Beta : Alpha 다음의 버전으로 주로 테스트를 위해 일반에 공개된다. 심각한 버그가 있을 수 있다. 

* CTP(Community Technology preview)Beta 버전과 비슷한 개념으로 관련 커뮤니티에 테스트를 진행하기 위한 용도로 배포된다. 인터넷 상에 Beta 대신에 공개되기도 한다. 

* RC(Release Candidate) : 출시 후보 버전으로 심각한 버그는 없고, 작은 버그 수정을 위해 공개된다. 

* Preview: RTM 바로 이전 버전으로 정식 출시 직전에 테스트 버전으로보면 된다. 

* RTM(Release To Manufacture) : 컴퓨터 제조 업체에 공급할 최종 완료된 버전으로 일반 소비자에게 판매할 버전보다 약간 먼저 출시된다. 하지만 일반 소비자용 정식 버전과 기능적차이는 없으므로 정식 버전이라고 생각 하면 된다. 

* Retail : RTM 버전을 DVD로 제작한 버전으로, 시장에서 소매로 판매되는 버전이다. 

위 버전을 모두 공개하는 것은 아니며, 회사에서 소프트웨어 특성상 필요한 것을 공개한다고 보면 된다. SQL Server 2012의 경우, CTP1 > CTP3 > RC0 > RTM의 순서로 인터넷에 공개 하였다. 







이 SQL Server 2012는 이전 버전에 비해서 향상된 많은 기능이 있으며, 특히 대용량 데이터베이스를 운영하기 위한 기술들이 많이 포함되었다. 하지만 이 책의 독자는 SQL Server 를 처음 다루거나 많이 사용한 적이 없으리라 생각한다. SQL Server 에 생소한 독자에게 SQL Server 2012에서 나아진 기능들을 나열해도 감흥이 오지 않을 것 같다. 게다가 알지도 못하는 부분이 첫 장에 나오면 학습에 대한 효과는 오히려 반감될 수 있다. 

이번 내용은 SQL Server를 처음 배우는 독자에게는 크게 중요한 것은 아니므로, 기존 SQL Server 사용자가 아니라면 간단히 훑어보는 정도로 보아도 좋다. 그리고 2장부터 나오는 설치를 진행하고, 이후에 어느 정도 SQL Server에 익숙해지면 이번 내용을 다시 편안하게 읽는 것이 더 좋은 학습 방법일수 있다. 그렇게 되면 SQL Server2012가 기존 버전에 비해 어떠한 좋은 기능이 있는지 자연스럽게 익힐 수 있으리라 생각한다. 

이 책의 전체 구성은 SQL Server 2012의 특징만 나열하는 것이 아니라 데이터베이스를 처음 접하는 독자가 데이터베이스를 학습하기 위한 도구로써 SQL Server2012를 사용하는 것에 초점을 맞추었다. 그리고 SQL Server 2012의 새로운 특징은 필요할 때마다 책 전체에서 자연스럽게 설명하여 독자가 SQL Server 2012뿐 아니라 DBMS에 최대한 부담 없이 접근할수 있게 했다. 





* SQL Server의 개요와 변천사
지금 이 책을 보는 독자라면 SQL Server(에스큐엘 서버 또는 시퀄 서버라고 읽으면 된다)가 무엇인지 공부하고자 하는 독자이거나, 학교에서 강의를 위해 보고 있는 독자, SQL Server를 사용해 본 일부 실무자들이 대부분일 것이다. SQL Server 를 처음 접해보든지, 이미 어느 정도 SQL Server를 사용해 보았든지, 아무튼 SQL Server가 무엇인지 알고 싶거나 SQL Server와 관련된 일을 하고 있을 것이다. 

SQL Server는 마이크로소프트 사에서 제작한 데이터베이스 관리 소프트웨어다. 데이터베이스 관리 소프트웨어(DBMS, DataBase Management System)는 한마디로 대량의 데이터를 관리해주는 소프트웨어라고 생각하면 쉬울 것이다. 

이처럼 DBMS는 대량의 데이터를 관리하기 위한 소프트웨어이며, SQL Server는 많은 DBMS 소프트웨어 중 마이크로소프트 사에서 제작한 툴이다. 우선은 SQL Server의 변천사를 간단하게 알아보도록 하자 다음 [표1-2]는 SQL Server의 변천을 버전별로 간략하게 나타낸다.

* SQL Server의 보편적인 특징
다음 내용은 SQL Server의 보편적인 특징을 요약한 것이다. SQL Server 2005, 2008, 2008 R2, 2012에 공통되는 특징으로 보면 된다. 

필자가 바라보는 SQL Server 의 버전별 차이는 다음 [그림 1-8]과 같다. 

SQL Server 2008(R2은 포함)은 SQL Server 2005에서 데이터베이스 내부적인 기능이 추가되고 확장된 개념으로생각하면 된다. 즉 SQL Server 2008은 SQL Server 2005의 기능을 그대로 포함하면서 많은 내부 기능이 추가된 것이다. 하지만 SQL Server 2012는 데이터 베이스 내부적인 기능보다는 외부 확장 기능이 더욱 많이 추가되었다. 이러한 외부 확장 기능은 주로 SQL Server 2008 R2 에서 소개되기 시작해서 SQL Server 2012에서 완성되었다고 볼 수 있다. 

SQL Server 2008, 2008 R2, 2012의 공통적인 특징을 요약하자면 신뢰성(Trusted), 생산성(Productive), 지능적(Intelligent) 세 가지다
이제 각각이 어떠한 세부적인 기능을갖는지 알아보자. 




※ 신뢰성
SQL Server 2008, 2008 R2, 2012는 이전 버전에 비해서 최고 수준의 보안과 신뢰성 (Trustability)을 제공하며 시스템의 규모에 따른 신축성을 제공한다. 

1. 정보의 보호 측면
* 투명한 데이터 암호화
SQL Server 자체에서 데이터 파일 및 로그파일의 암호화를 수행해 줌으로써 허가되지 않은 사용자로부터 데이터를 보호해 줄 뿐 아니라 만약 물리적으로 파일이 유출되더라도 파일로부터 데이터를 확인할 수 없게 암호화해 준다. 이를 투명한 데이터 암호화 (TDE : Transparent Data Encryption)이라고 부른다. 

* 키 관리 기능의 확장
SQL Server는 암호화 및 키 관리를 위한 종합적인 솔루션을 제공함으로써 ,데이터 기밀 유지에 대한 전반적인 문제를 예방할수 있다. 

* 감사 
DDL 문으로 감사(Auditing)를 설치하고 관리하면서 더욱 종합적인 데이터의 감사를 할 수 있다. 



2. 고가용성 측면
* 데이터베이스 미러링 기능을 통해 저렴한 고가용성(High Availability)데이터베이스를 구현할 수 있다. 

* 로그 스트림 압축
미러링에서 상호 전달되는 로그 스트림(Log Stream)이 자동으로 압축되어서 네트워크 상에 전달되는 양이 줄어든다. 결국 최종적으로 미러링의 응답속도가 빨라져 효율적으로 운영할 수 있다. 

* 데이터 페이지의 자동 복구
데이터 페이지 오류(오류번호 823, 824)문제를 데이터베이스 미러링을 구성할 때는 상호 자동 복구된다. 



3. 시스템 중지 없이 CPU 추가
CPU 및 메모리를 SQL Server가 작동중인 상태에서 추가할수 있게 되었다(Hot Add CPU) 이는 현재의 서비스를 중단하지 않은 상태로시스템 자원을 확장할수 있음을 의미한다. 



4. 성능과 확장성측면
* 리소스 관리자
리소스 관리자(Resource Governor)는 특정 업무가 하드웨어 자원을 모두 차지하는 것을 방지한다. 그럼으로써 실수 또는 고의의 잘못된 사용자 쿼리에 의해서 전체 시스템에 문제가 생기는 것을 예방할 수 있다. 

* 예측 가능한 쿼리 플랜
쿼리 플랜(Query Plan)을 잠가 놓을 수 있는 기능을 사용함으로써 서버를 업그레이드 하거나 새 제품을 배치하더라도 모든 하드웨어에 걸쳐 안정적인 쿼리 플랜을 실행할 수 있다. 

* 데이터 압축
데이터 압축이 개선되어 데이터 저장 공간의 크기를 줄일 수 있으며, 효율성 측면에서 도 성능 저하를 최소화 하였다. 

* 백업 압축
데이터를 백업할 때 압축 옵션을 제공함으로써, 백업되는 데이터의 크기가 획기적으로 줄었을 뿐 아니라 작은 데이터 전송으로 인해 백업 속도 또한 상당히 빨라졌다.




※ 생산성
SQL Server는 관리를 위한 시간과 비용을 최소화하여 생산성(Productive)을 높일수 있게 지원한다. 개발자는 .Net Framework 및 Visual Studio를 활용하여 응용프로그램을 손쉽고 강력하게 개발할수 있다. 

1. 관리성
* 정책 관리 
정책 관리(Policy-Based Management)는 하나 이상의 SQL Server 인스턴스를 관리하기 위한 정책 기반 시스템이다. 이를 통해서 시스템 구성 정책을 준수하고 나아가 시스템 관리의 일관성 및 효율성을 추구하게 된다. 

* 효율적인 설치
SQL Server는 설치 방식을 더욱 능률적이고 효율적으로 개선하였다. 

* 데이터 컬렉션
SQL Server의 성능과 관련된 데이터를 수집하는 기능(Data Collection)을 수행한다. 즉 SQL Server에 언제 무슨 일이 일어났는지에 대한 기록을 모은다. 이를 통해서 관리자는 시스템의 문제점을 파악하고, 자원 할당 등의 근거로 활용할 수 있다. 


2. 빠른 개발 환경 지원
* ADO .NET Entity Framework
개발자들은 데이터베이스의 열과 테이블을 대상으로 직접 프로그래밍 작업을 수행하는 것이 아니라, 고객 또는 주문 등과 같은 높은 수준의 추상적 엔티티를 사용하여 데이터를 표현함으로써 프로그래밍을 생산적으로 하고 싶어 한다. ADO .NET Entity Framework에서는 개발자들이 그러한 엔티티와 관련하여 관계형 데이터를 대상으로 프로그래밍을 수행할 수 있다. 

* LINQ(언어 통합 쿼리)
개발자들은 기존의 SQL 구문 대신에 C#이나 VB.NET 등의 프로그래밍 언어를 이용하여 데이터에 대한 쿼리를 수행할 수 있다. 

3. 다양한 데이터 형식의 지원
*DATETIME/DATE/TIME
SQL Server는 다양한 날짜 및 시간 데이터 형식을 지원한다. 

* HIERARCHY ID
HierachyId는 계층 트리의 노드를 대표하는 값을 저장할 수 있다. 

* FILESTREAM 데이터와 FILETABLE
대량의 바이너리 데이터가 데이터베이스내부가 아닌 윈도의 NTFS 파일 시스템에 직접 저장될 수 있게 해준다. 이를 통해 저장 효율이 획기적으로 향상되었다. 

*통합되고 뛰어난 성능의 전체 텍스트 검색
SQL Server 내부에 통합된 전체 텍스트 검색은 뛰어난 성능을 보장한다. 

* 스파스 열
스파스 열(Sparse Column)은 수많은 null값을 가진 테이블이 저장 공간을 그다지 많이 차지하지 않고 SQL Server 데이터베이스에 저장될 수 있게 한다. 

* 큰 사용자 정의 타입
SQL Server UDT(사용자 정의 형식)에 대한 8KB 제한이 없어져서 사용자들이 UDT 를 원하는 크기로 늘릴수 있다. 

* 공간 데이터 타입
지리정보시스템(GIS)에서 사용되는 Spatial 데이터 형식을 제공한다. 이를 통해 공간 데이터의 저장을 효율화하고, RDBMS의 제한을 뛰어넘게 되었다. 




※ 지능적
SQL Server는 사용자에게 지능적 정보를 제공하는 포괄적인 솔루션을 제공한다. 

1. 각종 데이터의 통합
* 테이블의 파티션 분할
SQL Server는 파티션 분할된 대용량 테이블에서의 성능을 높였다. 

* 스타 조인 쿼리 최적화
SQL Server의 스타 조인 쿼리 최적화 기능은 데이터 웨어하우스 조인 패턴을 인식함으로써 쿼리 응답 시간을 줄였다. 

* GROUPING SETS
GROUPING SETS는 서로 다르게 그룹화된 행의 UNION ALL과 같은 단일 결과 세트를 만들어내어 집계 쿼리 및 보고가 더욱 빠르고 쉽게 이루어지게 한다. 

* 변경 데이터 추적
SQL Server는 데이터 변경 추적(CDC: Change Data Capture)기능을 제공한다. 이는 변경된 데이터를 추적하고 데이터의 일관성을 보장해주는 역할을 할 수 있다. 

* MERGE SQL 구문
MERGE SQL 문의 도입으로, 개발자들은 행의 존재 여부를 확인하고 삽입 또는 업데이트를 실행하는 등 일반적인 데이터 웨어하우스 작업 상황을 더욱 효과적으로 처리할 수 있다. 


2. 관련 보고서 제공
* Reporting Services의기능
Reporting Services 기능을 통해 사용자가 관련 정보에 즉시 액세스하여 해당 업무 분야의 현황을 깊이 있게 파악할 수 있고, 이를 통해 보다 정확하고 신속하게 합리적인 의사 결정을 내릴 수 있다. 

* 마이크로소프트 오피스와 통합
SQL Server Reporting Services가 마이크로소프트 오피스 SharePoint와 긴밀하게 통합되어 엔터프라이즈 보고서와 기타 비즈니스 현황 자료를 중앙에서 배포하고 관리할수 있다. 






※ SQL Server 2012의 새로운 특징
SQL Server 2012에서는 많은 기능들이 추가되고 기존의 기능들도 많이 발전하였다. 그중 마이크로소프트에서 강조하는  SQL Server2012의 특징은 다음 3가지다. 

* 미션 크리티컬 비즈니스 환경에서 신뢰도 보장

* 현상 돌파의 통찰력 제공

* 상황에 맞는 클라우드 활용

각각에 대해서 살펴보겠다. 

* 다음 설명된 내용은 마이크로소프트에서 주장(?)하는 SQL Server2012의 대용량, 대 기업적 사용에 관한 특징들이 요약된 것이다. 이러한 특징은 지금 학습하는 1권(기본편)의 내용과는 별로 관계가 없다. 그러므로 내용이 이해되지 않더라도 책의 학습과는 무관하니 대략 훑어보기만 해도 된다. 







※ 미션 크리티컬 비즈니스 환경에서 신뢰도 보장

1. 중요 업무에 필요한 가용성과 보호 기능
시간과 비용의 낭비 없이 서버에서 클라우드로 기업이 원하는 가동시간과 데이터 보호 기능을 제공한다. 

* SQL Server AlwaysOn이라는 새로운 고가용성 솔루션을 통해 문제가 발생했을 때 신속하게 장애 조치가 수행될 수 있게 지원한다. 

* 보조 데이터베이스 여러 개를 구성할 수 있는 옵션과 애플리케이션을 신속하게 장애조치 및 복구하는 기능을 통해 애플리케이션 가용성을 높이고 데이터를 보호한다. 

* 통합된 구성과 모니터링 도구를 이용하여 간편하게 고가용성 솔루션을 배포하고 관리한다. 



2. 초고속 성능
혁신적이며 예측 가능한 성능을 지원한다. 

* 새로운 ColumnStore index를 통해 스타 조인 및 유사 쿼리에서 최대 100배 향상된 속도를 제공한다. 

* 전체 텍스트 검색 기능을 획기적으로 개선했다. 

* 압축 기능을 통해 증가하는 데이터 저장 공간을 약 50%까지 줄일수 있게 되어 입출력 성능을 향상시켰다. 

* 리소스 관리자는 SQL Server의 다양한 작업 부하에 대해 일관된 성능을보장한다 

* 친숙하고 사용이 편리한 관리 도구를 이용하여 쿼리 실행 계획을 분석한다. 

3. 조직 보안 및 규정 준수
기본적으로 제공되는 보안 및 IT 제어 기능을 제공한다 

* 기본 암호화 기능으로 데이터를 보호한다. 

* 감사 복원, 필터링, 사용자 지정 감사 기능으로 감사에 대한 관리 효율성을 높인다. 

* 데이터베이스 스키마의 복잡성을 줄였다. 

* 사용자 정의 서버 역할(User-Defined Audit)은 데이터 액세스에 할당된 권한을 간편하게 관리한다. 

* 구성 관리자를 통해 필요한 서비스만을 활성화하여 공격 부위를 최소화한다. 





※ 현상 돌파의 통찰력


1. 신속한 데이터 탐색 및 시각화
신속한 데이터 탐색 및 시각화로 대량 데이터를 조사하여 깊은 통찰력을 제공한다. 

* 대화형 그래픽 인터페이스를 제공하는 빠른 속도의 Power View를 사용하여 전사적으로 통찰력 있게 검색하고 공유한다. 

* 고급 분석 기능을 내장한 PowerPivot으로 많은 데이터를 빠른 속도로 처리할 수 있다. 


2. 확장 가능한 Managed Self Service BI
SharePoint 협업으로 인텔리전스 사용 범위를 기업 전체로 확대하였다. 

* BI Sematic Model은조직 들의 소규모 BI 솔루션을 큰 조직적 BI 요구에 맞게 확장할수 있게 했다. 

* IT 부서가 최종 사용자 활동 및 데이터 소스 사용량을 모니터링하고 서버에서 성능 메트릭을 수집할 수 있게 지원한다. 

* 뛰어난 최종 사용자 분석 도구를 배포하는 SQL Server는 기존 OLAP와 보고 큐브를 간편하게 구축하고 관리함으로써 종단 간 BI를 지속적으로 제공한다. 



3. 신뢰할수 있는 일관된 데이터
전체 비즈니스 데이터를 한눈에 파악할 수 있게 하고 통합, 정제 및 관리를 통해 데이터 신뢰성을 보장한다. 

* SQL Server 2012에 내장되어 있는 SSIS, MDS, DQS를 활용하여 Windows Azure Marketplace DataMarket을 비롯하여 여러 곳으로 분산되어 있는 데이터 소스를 통합, 관리한다. 

*SSIS는 모든 규모의 조직들에 정보 관리 업무에서 높은 효율성 및 생산성으로 정보 관련 업무를 조작할 수 있게 해줌으로써 쉽게 데이터 통합을 시작할 수 있게 지원한다. 

* 새로운 SQL Server Data Qulity Service(DQS)는 고객들이 조직적으로 지식을 활용할수 있게 데이터 품질을 보증한다. 

* Master Data Service(MDS)는 데이터 통합 작업에 이용되는 마스터 데이터 구조를 손쉽게 관리하며, 엑셀용 MDS 추가 기능을 통해 최종 사용자가 중앙 데이터를 간편하게 관리하고 유지할 수 있다. 



4. 완벽한 데이터 웨어하우스 솔루션
데이터 웨어하우스는 적은 비용으로 대규모 확장을 지원하여 확장성과 유연성을 제공한다. 

* 적은비용으로 뛰어난 성능을 제공하여 데이터 웨어하우스 비용이 절감된다. 

* 15,000개 파티션으로 확장되는 분할 테이블 및 Remote Blob Storage와 같은 기본 데이터베이스 기능들을 이용하여 데이터 웨어하우스를 확장한다.

* 최대 256개 코어를 지원하는 확장성을 제공한다. 

* 수백 TB 크기로 확장 가능한 Parallel Data Warehouse와 MPP(대량 병렬 처리)아키텍쳐를 통해 높은 성능을 유지한다. 






※ 상황에 맞는 클라우드 활용

1. 주문형(On-Demand)확장
유연한 배포 옵션을 통해 서버에서 클라우드로 확장이 가능하다. 

* SQL Server 2012와 상호 운용이 가능한 배포 옵션을 제공하는 마이크로소프트 클라우드 제품을 사용하면 운영 효율성을 높힐 수 있다. 

* 기존 서버, 클라우드에서 사용할 수 있는 공통 아키텍처를 활용하여 배포 환경의 한계를 넘어서는 확장이 가능하다. 

* SQL Server 2012와 SQL Azure의 마이크로소프트 사설 및 공용 클라우드 제품들을 이용하면 필요할 때 가상화된 용량과 셀프 서비스 어플라이언스(Appliance)및 사용량 측정이 가능하다. 

* 클라우드 데이터베이스 서비스인 SQL Azure를 통해 소규모에서 대규모로 확장할 수 있다. 


2. 빠른 솔루션 출시 기간
어플라이언스와 사설/공용 클라우드 서비스를 통해 복잡성을 줄이고 솔루션 출시 기간을 단축할 수 있다. 

* SQL Server 어플라이언스는 모든 하드웨어 및 소프트웨어 구성 요소를 설계, 조정 및 테스트하는데 소요되는 시간을 줄임으로써 솔루션 출시 기간을 단축한다. 

* 다양한 규모의 데이터 웨어하우징, 비즈니스 인텔리전스, 사설 클라우드, 통합 및 트랜잭션 프로세싱을 위한 어플라이언스를 이용하여 우선 순위가 높은 작업들을 신속하게 처리한다. 

* SQL Azure를 이용하면 물리적 관리 또는 인프라 투자 없이 수 분 내에 데이터베이스를 준비하고 제공할 수 있다. 



3. 최적화된 생산성
단일 도구를 이용하여 서버와 클라우드 전반에 걸쳐 IT 팀과 개발자의 생산성을 최적화 한다. 

* 차세대 웹, 엔터프라이즈 및 데이터 인식 모바일 애플리케이션을 구축하는 개발자들은 SQL Server Data Tools에서 제공하는 통합 데이터 베이스를 통해개발 수준을 높일 수 있다. 

* 같은 개발 도구, 프레임워크 및 T-SQL 기반 모델을 이용하여 데이터베이스 또는 비즈니스 인텔리전스 솔루션을 구축, 배포 및관리한다. 

* 단일 콘솔인 SQL Server Management Studio(SSMS)를 통해 서버, 사설 클라우드 및 공용 클라우드까지 데이터베이스 자산을 배포 및 확장하여 효율적으로 관리할 수 있다. 



4. 어디서나 모든 데이터를 확장
광범위한 개발자 기술을 통해 데이터의 범위를 서버와 클라우드 전체로 확장한다. 

* 데이터 센터와 클라우드의 데이터 베이스 간 양방향 데이터 동기화를 제공하는 클라우드 서비스 SQL Azure Data Sync가 새로운 시나리오를 지원한다. 

* 웹 표준에서 구축된 공개 프로토콜 OData를 통해 웹, 장치 및 데스크톱에서 사용자 경험을 향상시키는 일관된 공개 데이터 피드를 제공한다. 

* XML 데이터, 윈도 파일 및 공간 정보를 포함하여 관계형 데이터와 비관계형 데이터를 관리하고 확장한다. 

* .NET, C/C++,Java 및 PHP와 같은 다양한 플랫폼에서 임의의 업계 표준 API(ADO, NET, ODBC, JDBC, PDO 및 ADO)를 이용하여 SQL Server와 SQL Azure 애플리케이션에 연결함으로써 이중 환경으로 확장한다.

※ 여기서 잠깐
SQL Server Express 에디션은 마이크로소프트 사이트(http://www.microsoft.com/express/sql/download/)에서 무료로 내려 받아 사용할 수 있으며, 재배포도 가능하다. 또한 SQL Server Express with Advanced Services를 내려받으면 확장된 기능까지도 무료로 사용할수 있다. 이책은 Enterprise 에디션을 사용하지만 소규모나 데이터 베이스 입문하는 사람들이 무료로 사용하기에 Express 에디션은 꽤 쓸만한 소프트웨어다.
