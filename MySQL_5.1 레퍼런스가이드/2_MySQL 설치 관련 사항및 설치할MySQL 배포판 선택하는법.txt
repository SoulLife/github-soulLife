CHAPTER02 : MySQL 설치 및 업그레이드하기
이 장에서는 MySQL을 다운로드하고 설치하는 방법에 대해 설명을 한다. 우선 개략적으로 설명을 한 다음에 나중에 보다 자세한 설명을 하기로 하겠다. 이미 설치되어 있는 버전을 새로운 버전으로 업그레이드를 할 생각이라면 Section 2.11, MySQL 업그레이드하기를 참조하기 바란다. 

1. 설치할 MySQL이 사용 중인 플랫폼을 지원하는지 검사 : 모든 시스템이 MySQL을 구동 시키는데 적합한 것은 아니라는 사실을 알아 두기 바란다. 
* MySQL 엔터프라이즈 서버의 경우에는 다음의 링크에서 공식 지원 플랫폼을 확인한다. 
http://www.mysql.com/support/supportedplatforms.html

* MySQL 커뮤니티 서버의 경우에는 Section 2.1.1 MySQL 커뮤니티 서버가 지원하는 OS에서 지원 플랫폼을 확인한다. 


2. 설치할 배포판 선택 : 대부분의 MySQL 버전은 여러 가지 형태의 배포판 포맷으로 제공된다. 사용자는 바이너리(미리 컴파일한) 프로그램 또는 소스 코드를 사용해서 미리 패키지 된(pre-packaged)배포판을 선택할 수 있다. 이것이 의심스러울 경우에는 바이너리 배포판을 사용하면 된다. 또한 현재 개발 단계에 있는 제품을 보고 새로운 코드에 대해 도움을 주고자 하는 사람들을 위해 현재의 소스 트리에 접속할 수 있는 방법도 함께 제공을 한다. 어떤 버전 및 타입의 배포판을 사용할 지 결정하기 위해서는 Section2.1.2 설치할 MySQL 배포판 선택하기를 참조하기 바란다. 

3. 설치할 배포판 다운로드하기 : 다운로드 방법에 대해서는 Section2.1.3, MySQL 을 다운로드하는 방법을 참조한다. 배포판의 완성도를 검사하기 위해서는 Section2.1.4 MD5 체크섬 또는 GnuPG를 사용해서 패키지 상태 확인하기를 참조한다. 

4. 배포판 설치하기 : 바이너리 배포판을 사용해서 MySQL을 설치하기 위해서는 Section2.2 바이너리 배포판을 사용해서 MySQL을 설치하는 표준 방법을 참조한다. 현재 개발 단계 소스 트리 또는 소스 배포판을 사용해서 MySQL을 설치하기 위해서는 Section2.9 소스 배포판을 사용해서 MySQL 설치하기을 참조한다. 
설치하는 과정에서 어려움을 느끼는 경우에는 Section 2.13, OS 관련 노트를 참조하여 특정 플랫폼에 대한 문제를 해결하기 바란다. 

5. 설치후 필요한 설정 작업 하기 : MySQL을 설치한 후에는 Section 2.10 설치 후 설정 및 테스팅을 읽어보기 바란다. 이 섹션에서는 MySQL 서버가 올바르게 동작할 수 있도록 만드는 중요한 정보를 설명하고 있다. 또한 초기 MySQL 계정에 대한 보안 작업 방법도 설명을 하는데, 이 초기 계정들은 패스워드를 설정하기 전까지는 패스워드를 갖고 있지 않게 된다. 이 섹션은 MySQL을 바이너리 또는 소스 배포판을 사용해서 설치했는지에 따라 적용이 된다. 

6. MySQL 벤치마크 스크립트를 구동 시키고자 한다면 MySQL 용 Perl을 반드시 설치해야 한다. Section 2.15, Perl 설치 노트를 참조하기 바란다.




2.1 설치 관련 일반 사항
MySQL 설치 과정은 MySQL 엔터프라이즈 서버를 설치할 것인지또는 MySQL 커뮤니티 서버를 설치할 것인지에 따라서 달라진다. 설치 가능 플랫폼 리스트를 알아보기 위해서는 아래의 내용을 참조하기 바란다 

* MySQL 엔터프라이즈 서버의 경우에는 다음의 링크에서 지원 플랫폼 리스트를 확인한다: http://www.mysql.com/support/supportedplatforms.html
* MySQL 커뮤니티 서버의 경우에는 Section2.1.1, MySQL 커뮤니티 서버가 지원하는 OS를 참조한다. 

MySQL 엔터프라이즈 서버의 경우에는 메인 배포판과 함께 엔터프라이즈 인스톨러를 사용해서 원하는 서비스 팩 또는 핫픽스(hotfixes)를 설치할 수 있다. 엔터프라이즈 인스톨러에 아직 포함되지 않은 플랫폼의 경우에는 커뮤니티 서버 설치 지침을 따라서 설치한다. 

MySQL 커뮤니티 서버의 경우에는 메인 배포판과 함께 모든 핫픽스와 업데이트를 설치한다. 

* 바이너리 릴리즈 또는 소스 릴리즈를 다운로드 한 후에, 소스 코드를 사용해서 직접 MySQL을 설치한다. 
* 비트키퍼(Bitkeeper) 트리에서 MySQL을 추출한 후에 소스를 사용해서 MySQL을 구축한다. 비트키퍼 트리에는 가장 최신 버전의 소스 코드가 들어 있다. 

다음에 나오는 섹션에서는 배포판을 선택하고, 다운로드 하고, 검증하는데 필요한 정보를 설명할 것이다. 



2.1.1 MySQL 커뮤니티 서버가 지원하는 OS
이 섹션에서는 MySQL 커뮤니티 서버를 구동 시킬 수 있는 OS에는 어떤 것이 있는지를 설명한다. 

Important:
보다 완벽한 지원 OS 리스트를 얻고자 한다면 www.mysql.com에서 MySQL Server Supported Platforms을 참조하기 바란다. 


우리는 GNU Autoconf를 사용하기 떄문에 C++ 컴파일러가 있고, POSIX 쓰레드(thread)를 실행하는 모든 OS 시스템에 MySQL을 설치할 수 있다(쓰레드 지원은 서버를 위해 필요하다. 클라이언트 코드만 컴파일 한다면 C++ 컴파일러만 필요하다)

MySQL은 아래와 같은 OS 및 쓰레드 패키지 조합에서 성공적인 컴파일이 이루어졌다는 보고를 받고 있다. 

* 네이티브 쓰레드를 사용하는 AIX 4.x, 5.x Section2.13.5.3 IMB-AIX 노트 참조
* 아미가(Amiga)
* 네이티브 쓰레드를 사용하는 FreeBSD 5.x 및 그 이상 버전
* 네이티브 쓰레드를 사용하는 HP-UX 11.x, Section 2.13.5.2, HP-UX Version 11.x 노트 참조
* glibc 2.3을 사용해서 구축된 리눅스, Section 2.13.1 Linux 노트 참조
* Mac OS X, Section 2.13.2 Mac OS X 노트 참조
* NetBSD 1.3/1.4 인텔 및 NetBSD 1.3 Alpha, Section2.13.4.2 NetBSD 노트 참조
* 노벨 NetWare 6.0 및 6.5 Section 2.7, NetWare에 MySQL 설치하기 참조
* 네이티브 쓰레드를 사용하는 OpenBSD 2.5 MIT-pthreads 패키지를 사용하는 OpenBSD 2.5 이전 버전, Section 2.13.4.3, OpenBSD 2.5 노트 참조
* SPARC 및 x86에서 네이티브 쓰레드를 사용하는 Solaris 2.5 및 그 이상 버전 Section 2.13.3 Solaris 노트 참조
* Windows 2000, XP 그리고 Windows Server 2003, Section 2.3 윈도우에 MySQL 설치하기 참조
